# Output Formats Reference

This document describes the platform-specific formats generated by the importer.

## Claude Preferences (`claude-preferences`)

**Target:** Claude's Settings > Profile > "What would you like Claude to know about you?"

**Format:** Natural language sentences, ~500 characters max recommended.

**Example:**
```
I am the CTO of TechFlow, based in San Francisco.
My work involves building an AI-powered analytics platform; preparing for Series A.
Currently focused on: Series A pitch deck; investor materials.
Technical skills: Python, TypeScript, AWS, Kubernetes, Docker.
Domain expertise: machine learning, data analytics.
Communication preference: concise, direct responses.
I frequently write technical documentation and investor updates.
```

**Usage:** Copy and paste into Claude's profile settings.

---

## Claude Memories (`claude-memories`)

**Target:** Claude's memory system (via memory_user_edits tool or manual addition)

**Format:** JSON array of statement strings, max 200 chars each, max 30 items.

**Example:**
```json
{
  "memories": [
    "User is the CTO of TechFlow",
    "User is based in San Francisco",
    "User works with Python, TypeScript",
    "User is focused on Series A funding",
    "User prefers concise, direct communication"
  ],
  "count": 5
}
```

**Usage:** Add manually via Claude's memory interface or programmatically.

---

## System Prompt (`system-prompt`)

**Target:** Any LLM system prompt (Claude API, OpenAI, local models, etc.)

**Format:** XML-wrapped context block that can be prepended to any system prompt.

**Example:**
```
<user_context>
The following information about the user was extracted from their conversation history:

**Identity:** CTO of TechFlow; based in San Francisco
**Professional Context:** AI-powered analytics platform; Series A preparation
**Active Priorities:** Series A pitch deck; investor materials
**Technical Expertise:** Python; TypeScript; AWS; Kubernetes
**Domain Knowledge:** machine learning; data analytics
**Communication Preferences:** concise, direct responses

</user_context>
```

**Usage:** Prepend to your system prompt when calling any LLM API.

---

## Summary (`summary`)

**Target:** Human review before importing

**Format:** Markdown document with categorized facts and confidence indicators.

**Usage:** Review to verify accuracy before importing to target platform.

---

## Confidence Filtering

| Threshold | Score | What's Included |
|-----------|-------|-----------------|
| `high` | ≥ 0.8 | Only well-established facts (5+ mentions) |
| `medium` | ≥ 0.6 | Reliable facts (3+ mentions or explicit statements) |
| `low` | ≥ 0.4 | Include inferred facts (2+ mentions) |
| `all` | ≥ 0.0 | Everything extracted |

**Recommendation:** Start with `medium` for best balance of coverage and accuracy.
