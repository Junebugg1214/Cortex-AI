Plain-English explanation

What each subsystem is doing

Portable AI

This is how a Mind learns. Cortex scans existing context, ingests durable facts, and keeps runtime context synchronized without requiring you to paste the same setup over and over.

Brainpacks

Brainpacks are specialist cognition layers. Raw sources go in, Cortex compiles them into a small wiki, graph, claims, unknowns, artifacts, and mountable context.

Git for AI Memory

Cortex treats memory as something that should be reviewable and reversible. That is why branch, diff, merge, review, and rollback are core product behaviors, not extras.

Mounts and composition

Cortex does not send one giant instruction blob everywhere. It composes the right memory slice for Codex, Cursor, Claude Code, MCP, Hermes, OpenClaw, task, policy, and attached Brainpacks, then mounts it.

Runtime loop

What happens when someone actually uses Cortex?

1

Create or choose a Mind

This becomes the persistent memory container for a person, project, agent, or team.

2

Ingest base context

Teach the Mind direct facts, import existing context, or route classic portability commands through it.

3

Attach Brainpacks when needed

Add specialist knowledge packs with activation rules for specific tools or tasks.

4

Branch risky work

Create isolated lines of memory history so experiments or project-specific changes do not leak into the main state.

5

Compose and mount

Render the exact memory slice for Codex, Cursor, Claude Code, MCP, Hermes, or OpenClaw and materialize it into the environment.

CLI example

A concise end-to-end flow

$ cortex mind init marc --kind person --owner marc
$ cortex mind remember marc "I prefer concise, implementation-first responses."
$ cortex pack init ai-memory --description "Portable AI memory research" --owner marc
$ cortex pack compile ai-memory --suggest-questions
$ cortex mind attach-pack marc ai-memory --always-on --target codex --target hermes --task-term memory
$ cortex mind compose marc --to codex --task "memory routing"
$ cortex mind mount marc --to hermes openclaw codex cursor claude-code --task "launch prep"
Good fit

When Cortex makes sense

You need durable context across tools, want control over storage, care about reviewable memory changes, or need specialist knowledge that can mount selectively.

Not the pitch

What Cortex is not trying to be

It is not positioned as a hosted memory cloud or a generic multi-tenant SaaS. The strongest story today is local-first and operator-managed self-hosting.
